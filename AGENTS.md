# AGENTS.md — guidance for AI coding assistants

For Cursor, Aider, Roo Code, Continue, Cline, and similar tools. (Claude Code reads `CLAUDE.md` instead — it's the same content.)

## What this repo is

A player template for **Battle of Code**, a multiplayer territory-capture game. Each player forks this repo, runs `./scripts/setup.sh` to register a key, then writes their own bot in `bot/mybot/bot.py`.

## Where to edit

- **Bot logic**: `bot/mybot/bot.py`. This is the only file you should normally touch.
- **Reference**: `bot/example/`. **Read-only** — it tracks upstream.
- **SDK**: `bot/sdk/python/bocbot/`. **Read-only**.
- **CLI player**: `client/`. Edit only when fixing a bug.

## Where to read

- `docs/GAME-RULES.md` — capture / death / kill rules (source of truth).
- `docs/PROTOCOL.md` — wire protocol (every JSON message in/out).
- `docs/EXAMPLES.md` — short cookbook patterns.
- `bot/example/ai.py` — full deterministic reference strategy.

## Hard rules

- Never commit `keys/*.key`. Private keys are git-ignored, don't `git add -A` past the rule.
- Never push `keys/<login>.pub` to `main`. It lives on the `<github-login>` branch only.
- Never edit `bot/example/` or `bot/sdk/` — both pull from upstream.
- No "generated by AI" comments or commit trailers.

## Bot contract

```python
from bocbot import Bot

class MyBot(Bot):
    def decide(self, state) -> str:
        # state matches docs/PROTOCOL.md `state` message
        return "D"   # one of W A S D N

if __name__ == "__main__":
    MyBot().run()
```

`decide()` runs every tick. Keep it pure and fast — no IO, no network. Persist memory between ticks on `self`.

## Style

- Python 3.10+, type hints on public functions, snake_case names.
- Imports: stdlib, then third-party, then `from bocbot import ...`.
- No print statements — use `logging`.
- One concern per file. If `bot.py` grows past ~200 lines, split into `bot.py` + `strategy.py` and import.

## What's missing on purpose

- No CI. Don't add GitHub Actions unless asked.
- No tests. Add `pytest` if the user requests one.
- No bot orchestration. One process, one bot, foreground.
