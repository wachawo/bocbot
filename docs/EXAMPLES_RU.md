Короткие паттерны для `decide(state)`. Всё в этом файле предназначено для вставки в `bot.py` с последующей адаптацией — никакого SDK или внешней библиотеки кроме того, что `bot.py` уже импортирует.

`decide(state)` принимает dict `state` и возвращает одно из `"W" | "A" | "S" | "D" | "N"`. Вызывается ~10 раз в секунду. Держите её быстрой: без файлового I/O, сетевых вызовов, медленных импортов.

## Как избегать стен

Следующий шаг приземляется в `(me.x + dx, me.y + dy)`. Если эта клетка вне `[0, map_w) × [0, map_h)` — вы умираете с `reason="out_of_bounds"`. Выбирайте направление, у которого следующий шаг остаётся на карте.

```python
DELTAS = {"W": (0, -1), "S": (0, 1), "A": (-1, 0), "D": (1, 0)}

def decide(state):
    me, w, h = state["me"], state["map_w"], state["map_h"]
    cur = me["dir"]
    dx, dy = DELTAS.get(cur, (0, 0))
    nx, ny = me["x"] + dx, me["y"] + dy
    if 0 <= nx < w and 0 <= ny < h:
        return cur
    for d, (dx, dy) in DELTAS.items():
        if 0 <= me["x"] + dx < w and 0 <= me["y"] + dy < h:
            return d
    return "N"
```

`state["map_w"]` и `state["map_h"]` приходят из фрейма `welcome`, который обвязка сохранила до первого `state`.

## Как охотиться на других игроков

Игрока можно убить, когда у него `trail_len > 0` (его голова вне его собственной зоны). Шаг на **любую** клетку его trail убивает. Не нужно гнаться за головой.

`state["view"].trail` — это grid trail для view-окна. Найдите клетку trail врага рядом с собой и шагните на неё.

```python
def decide(state):
    me = state["me"]
    view = state["view"]
    trail = view["trail"]
    h, w = len(trail), len(trail[0])
    for d, (dx, dy) in DELTAS.items():
        nx, ny = me["x"] - view["x0"] + dx, me["y"] - view["y0"] + dy
        if 0 <= nx < w and 0 <= ny < h:
            owner = trail[ny][nx]
            if owner != 0 and owner != me["id"]:
                return d
    return me["dir"] or "D"
```

Избегайте лобовых столкновений: если враг доберётся до **вашего** trail за меньше tick'ов, чем вы до его — отступайте. Простой Manhattan-чек обычно достаточен — см. «расстояние» ниже.

## Как считать расстояние

- **Manhattan** (`abs(dx) + abs(dy)`) — дёшево, игнорирует препятствия. Подходит для «этот игрок рядом?» или «кто из этих ближе?».
- **Чебышёв** (`max(abs(dx), abs(dy))`) — полезно когда важна дальность обзора (view — это квадрат с вами в центре).
- **BFS** по grid'у — нужен только когда важна *достижимая* дистанция через свою зону vs в обход чужой.

```python
def manhattan(ax, ay, bx, by):
    return abs(ax - bx) + abs(ay - by)
```

View — это `view_w × view_h` клеток (по умолчанию 41×41). BFS внутри него дешёвый — сильно меньше миллисекунды. BFS по всей карте (1024×768 ≈ 786 K клеток) — нет; держите его внутри view, если только реально не нужна дальняя достижимость.

## Когда возвращаться домой

Две эвристики, отсортированные по простоте:

1. **Cap на длину trail.** Если `me.trail_len > MAX_TRAIL` — возвращайтесь в зону. `MAX_TRAIL = 80` — разумный старт.
2. **Бюджет расстояния.** Считайте Manhattan от головы до ближайшей клетки своей зоны. Если эта дистанция ≥ оставшегося survival-бюджета — возвращайтесь. «Бюджет» зависит от того, насколько близок ближайший враг и сколько клеток *вашего* trail он может достичь до того как вы достигнете *своей* зоны.

Стартовый бот в `bot.py` использует cap на длину trail, потому что про него легко рассуждать. Замените на rule по бюджету расстояния, когда посмотрите несколько проигрышей.

## Как читать scores

`state["scores"]` — список dict'ов, когда сервер его шлёт (примерно раз в 5 секунд; между этим поле отсутствует или содержит тот же объект). У каждого dict есть `pid`, `name`, `area`, `kills`, `deaths`.

```python
def top_rival(state, me_pid):
    scores = state.get("scores") or []
    others = [s for s in scores if s["pid"] != me_pid]
    if not others:
        return None
    others.sort(key=lambda s: -s["area"])
    return others[0]
```

Используйте это для приоритизации цели — если зона лидера граничит с вашей, откусить от него стоит больше, чем захватить пустое пространство.

## Как применять событие `captured` для fog-памяти

Обвязка в `bot.py` уже это делает, но полезно знать что:

`captured.cells` — список пар `[x, y]`, только что перешедших в вашу зону (trail-клетки + flood-filled-интерьер). Клиент раскрашивает их в fog-of-war память, чтобы клетки вне live-view всё равно выглядели правильно. Если вы строите свою memory map внутри `decide` — применяйте этот delta сразу же по приёме события.

## Анти-паттерны

- **Прокладка длинного trail полностью внутри одной чужой зоны.** Самая часто упускаемая причина смерти — `trapped_in_zone` (см. [RULES.md](RULES_RU.md) § 7). Ходите по *краю* чужой зоны или через несколько разных зон, но не внутри одной.
- **Реакция на каждый state-фрейм.** State приходит на 10 Hz. Если меняете направление каждый tick, 1-tick cooldown означает что сервер дропает большинство — и вас «трясёт». Решите цель, затем коммитьтесь до её достижения или инвалидации.
- **Попытка убежать от более близкого врага.** Скорость у всех одинаковая. Если он ближе к краю вашей зоны, чем вы — не убежите; сворачивайте в другой вектор пока он не отрезал.

## Откуда импорты в шаблоне бота

`bot.py` импортирует:

- `cryptography` — для подписи `hello`.
- `websockets` — для WebSocket-транспорта.
- `os`, `time`, `json`, `traceback` — stdlib.

Нет пакета `bocbot`, нет `from bocbot import Bot`, нет SDK. Весь бот — это файл перед вами.
